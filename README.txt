    Конечная точка                    Описание

/auth/users/                Зарегистрировать нового пользователя, запрос: POST, без авторизации, JSON:
                            {
                            "username": "Vasja",
                            "password": "passVasja",
                            "email": "Vasja@server.com"
                            }


/auth/users/me/             Получить зарегистрированного пользователя, запрос GET, авторизация JWT.


/auth/jwt/create/           Создать JWT для пользователя, запрос: POST, аторизация через имя и пароль в JSON:
                            {
                                "username":"Vasja",
                                "password": "passVasja"
                            }


/auth/jwt/refresh/          Получить новый JWT по истечении времени жизни ранее сгенерированного, запрос POST,
                            авторизация через токен refresh в JSON:
                            {
                                "refresh": "eyJ0eXAiOiJKV1QiLCJhbGc..."
                            }


/api/tickets/               Получить все доступные пользователю тикеты, запрос GET, авторизация JWT.

                            Создать новый тикет, запрос POST, авторизация JWT, JSON:
                            {
                                "new_ticket": {
                                    "title_ticket": "Новая заявка",
                                    "text": "Описание"
                                }
                            }
                            При создании нового тикета отправляются письма автору и администраторам


/api/tickets/id/            Подробный вид тикета, запрос GET, авторизация JWT.

                            Создать новое сообщение, запрос POST, авторизация JWT, JSON:
                            {
                                "new_message": {
                                        "text": "ответ"
                                }
                            }
                            При создании нового сообщения отправляются письма автору и администраторам


                            Изменить статус тикета, запрос PUT, авторизация JWT, JSON :
                            {
                                "ticket": {
                                        "status_ticket": "H" # допустимые значения: 'O', 'H', 'R', 'C'
                                }
                            }
                            При изменении статуса тикета отправляется письмо автору тикета
